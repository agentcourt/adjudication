The question is whether `arb` wants advocacy to remain role-bound even when investigation turns against the side, or whether it wants advocacy to collapse into a neutral report at that point. The current prompt tries to hold both ideas at once. That is what fails.

I see three coherent designs.

The first design keeps strict role advocacy. Plaintiff must still file the strongest truthful `YES` case, even if that case is weak. In that design, the argument prompt should say exactly that: when investigation reveals serious adverse facts, present them, narrow the claim, and then state the best surviving basis for your side. The filing can say, in substance, “the present record leaves plaintiff with only this narrow theory.” That preserves the adversarial structure. It also fits the engine, because arguments already cannot pass. The weakness is that some models may still hesitate when the best surviving basis is very thin, unless the prompt gives an explicit example of such a filing.

The second design allows a merits-phase pass. If a side concludes that truthful investigation leaves no non-frivolous argument, it may pass. That resolves the tension cleanly at the interface level. It also changes the procedure more substantially. In an adversarial system, letting a side decline its main argument phase changes the structure of the record and the strategic shape of the case. This is not just a prompt fix. It is an engine and proof change.

The third design distinguishes advocacy from record candor more formally. The attorney must file in arguments, but the filing may expressly conclude that the assigned side has not met its burden on the current record. In effect, the attorney’s duty becomes: investigate, then report the side’s strongest available case, including the conclusion that the case may fail. That is close to the first design, but more explicit. It gives the model a legitimate end state when the best truthful advocacy is materially concessionary. The filing still exists, so the procedural rhythm stays intact. The downside is conceptual. Some people will say that this no longer looks like ordinary advocacy, because the lawyer is authorized to tell the council that its own side likely loses.

My inclination is toward the first design, but only if it is made explicit. `arb` is built as an adversarial procedure. The attorney should still argue for the assigned side. The prompt should tell the attorney what that means when the evidence turns: disclose the adverse point, state why it is adverse, identify any remaining pro-side theory that survives honest scrutiny, and file that narrowed argument. If nothing survives except a formally possible but weak theory, the attorney should still say so and present it. That keeps the adversarial shape and avoids adding a pass path to arguments.

The reason I do not favor the second design by default is that it lets the model evade the hard part of advocacy. The hard cases are exactly where counsel must distinguish “the best truthful case for my side is weak” from “I will not file one.” If the system wants adversarial testing, it should collect that weak but disciplined filing. Right now the model is not doing that because the prompt never states it clearly.

So the decision point is this: in `arb`, do you want an attorney in arguments to be allowed to say, in effect, “on the current record, my side likely loses, but here is the narrowest argument that still survives”? If yes, I think we can stabilize `ex6` without changing the engine. If no, and you instead want a side to be able to decline merits argument entirely, then the correct fix is procedural, not merely prompt-level.
